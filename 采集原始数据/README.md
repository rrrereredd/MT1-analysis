## 摘要

本报告以 `F:\1\依怕热古丽\采集原始数据` 中的过程采集数据为对象，围绕发电负荷、AGC 控制、一次风量、二次风量、煤粉浓度和阀位反馈等变量开展分析。项目重点不在于建立高复杂度模型，而在于识别变量类别、整理时间序列、计算滞后相关并输出适合作业提交的分析结果。

## 项目背景

工业过程采集数据通常具有以下特点：变量多、采样频率不一致、名称不统一、联动关系强。如果直接查看每个 CSV 文件，很难快速得到规律。因此本项目将文件按类别进行识别和汇总，再进一步观察 AGC 与负荷响应、风量与煤粉浓度、阀位与反馈量之间的关系。

## 数据概况

- 数据目录下共有 369 个 CSV 文件
- 字段统一为 `Server、Tagname、Value、Timestamp、Questionable、Annotated、Substituted`
- 时间跨度主要覆盖 2021 年 7 月下旬到 2021 年 8 月下旬
- 目前按文件名识别后，大致可分为：
  - 煤粉浓度类：64 个
  - 二次风类：41 个
  - 一次风类：28 个
  - 阀位类：27 个
  - AGC 类：2 个
  - 发电负荷类：4 个
  - 其他类：203 个

这表明原始数据中既有明确的过程变量，也有大量需要进一步确认含义的测点。

## 分析目标

1. 识别文件所属的业务类别
2. 整理时间轴并统一时间格式
3. 输出各类变量的统计概况
4. 分析 AGC 与负荷之间的滞后关系
5. 分析风量、煤粉浓度和阀位反馈之间的关联性
6. 为后续建模或统一管理平台提供基础分析结果

## 方法与实现

项目采用 Python 完成分析，使用 `pandas` 进行数据读取和汇总，使用 `matplotlib` 和 `seaborn` 生成图表。当前实现包括：

- 单文件分析
- 目录批量分析
- 按文件名分类
- 类别汇总表生成
- AGC 与发电负荷的滞后相关分析

## 主要结果形式

- `acquisition_summary.json`
- `metric_summary.csv`
- `top_metrics.png`
- `lag_correlation.csv`
- `lag_correlation.png`

这些结果可直接用于报告撰写，也可以作为后续深入分析的基础数据。

## 结论

从当前样本看，采集原始数据覆盖了机组控制和过程量的多个重要层面，具备较好的联动分析价值。其中，AGC 和发电负荷适合用于响应时滞分析，一次/二次风量和煤粉浓度适合用于系统联动分析，阀位反馈则适合用于控制回路观察。整体上，该数据集适合进一步扩展到预测、诊断和统一管理平台方向。

## 运行示例

### 1. 单文件分析

```bash
python src/acquisition_analysis.py "F:\1\依怕热古丽\采集原始数据\AGC控制指令 361.csv"
```

该命令会输出单个过程采集文件的统计摘要，适合用于检查变量是否能正常读取。

### 2. 目录批量分析

```bash
python src/acquisition_analysis.py "F:\1\依怕热古丽\采集原始数据"
```

该命令会遍历目录中的所有 CSV 文件，并输出各文件的统计摘要和类别识别结果。

### 3. 生成完整报告

```bash
python src/acquisition_analysis.py "F:\1\依怕热古丽\采集原始数据" --report-dir output
```

执行后会生成类别汇总表、滞后相关表和图表文件，适合直接放入提交材料。
