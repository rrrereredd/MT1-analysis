# 项目一使用说明

## 目录

- 4. 输出文件用途
- 5. 目录结构建议
- 6. 常见问题

## 1. 单文件分析

```bash
python src/weather_analysis.py "F:\1\MT1\气象原始数据\data\201901.csv"
```

此命令用于快速检查单个气象文件的结构和统计信息。

## 2. 目录批量分析

```bash
python src/weather_analysis.py "F:\1\MT1\气象原始数据\data"
```

此命令会遍历目录中的所有 CSV 文件，并输出每个文件的分析摘要。

## 3. 生成完整报告

```bash
python src/weather_analysis.py "F:\1\MT1\气象原始数据\data" --report-dir output
```

执行后会生成：

- `weather_summary.json`
- `weather_city_summary.csv`
- `temperature_trend.png`
- `correlation_heatmap.png`

## 4. 输出文件用途

- `JSON`：适合保存结构化摘要
- `CSV`：适合后续二次处理
- `PNG`：适合放入报告或答辩材料



## 5. 目录结构建议

为了便于打包提交，建议项目一的目录结构保持如下形式：

```text
项目一/
```

## 6. 常见问题

### 6.1 中文文件名打不开

通常是终端编码或路径输入不正确导致的。请确认命令行中输入的是完整绝对路径，并尽量使用双引号包裹路径。

### 6.2 图表没有生成

请确认 `matplotlib` 和 `seaborn` 已正确安装，并检查目标目录是否有写入权限。

### 6.3 输出文件为空

如果输入目录中没有 CSV 文件，或者字段无法正确识别，脚本可能会输出空摘要。这时需要先检查原始文件是否存在、是否损坏、字段名是否与预期一致。

## 7. 建议的提交顺序

1. 先运行测试确认源码正常
2. 再运行报告生成命令输出结果文件
3. 最后把文档、代码和结果一起打包
